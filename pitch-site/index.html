How It Works

Publish once.
Install everywhere.

From hook author to Claude user in three commands. The same workflow developers already know from npm — applied to the most powerful AI automation primitive.

1

Build & Publish

Write your hook, add a hook.json manifest, open a PR to the registry. CI validates schema, runs static security analysis, merges to index.

hookpm publish
2

Discover & Verify

Users search by name, tag, event type, or capability. Every hook shows its security status, capability requirements, and author reputation before install.

hookpm search security
3

Install & Run

One command downloads, verifies SHA-256 integrity, and atomically merges into ~/.claude/settings.json. Claude Code picks it up immediately.

hookpm install <name>
⚡

PreToolUse Hooks

Run before Claude executes any tool call. Block dangerous bash commands, enforce least privilege, log every file touched.

✅

PostToolUse Hooks

React to what Claude just did. Auto-format code, run linters, update changelogs, trigger downstream pipelines.

🔔

Notification Hooks

Intercept every Claude notification. Route to Slack, trigger alerts, log to audit systems, enforce compliance rules.

Competitive Moat

Why hookpm wins
and stays won.

The defensibility isn't the code — it's the ecosystem lock-in. First-mover advantage, author relationships, and schema ownership compound into a durable moat.

Schema Ownership

hookpm defines hook.json — the contract every hook must satisfy. Whoever owns the schema owns the standard. The Zod schema is open-source, but the registry enforces it.

Author Lock-in

Once 500+ authors have published hooks, built audiences, and earned certified badges, migration cost is enormous. This is the GitHub effect — no one moves a repo after it has stars.
